# new-project

ユーザーへのヒアリング → 設計 → フォルダ・ドキュメント整備 の順に進める。

## フェーズ 1: 要件ヒアリング

`AskUserQuestion` ツールを使い、設計漏れが発生しないよう不明点が解消するまで繰り返し質問する。

最初に確認すること:
- **用途・種類**: Web アプリ(API+UI) / CLI ツール / デスクトップアプリ / ライブラリ
- **具体的にやりたいこと**: どんな問題を解決したいか

選択肢がある場合（UIフレームワーク等）は **preview 付き**で Pro/Cons を提示してユーザーに選ばせる。

## フェーズ 2: 設計

**architect エージェント**に要件を渡して以下を設計させる:
- 技術スタックの選定と根拠
- ディレクトリ構造（ファイル名・役割付き）
- 主要コンポーネントの責務・API 一覧
- UX フロー（フロントエンドがある場合）
- 実装順序・検証方法
- 使用ライブラリの最新安定バージョンと主要設定
- 全コマンドは `Taskfile.yml` に集約する（npm scripts や cargo コマンドを直接叩かず `task xxx` で統一）

## フェーズ 3: ドキュメント整備

### `.claude/plan.md`（詳細設計書）

```markdown
# 計画: {プロジェクト名}

## Context
## 技術スタック
## ディレクトリ構造
## 主要コンポーネント・API
## DB スキーマ（必要な場合）
## UX フロー（フロントエンドがある場合）
## 依存パッケージ
## 実装順序
## 検証方法
```

### `CLAUDE.md`（プロジェクト概要）

詳細設計は書かない。概要・構成・コマンドのみ。

```markdown
# {プロジェクト名}

{1〜2行のアプリ概要}

## 概要
## 構成
## コマンド
## 開発環境セットアップ
```

## フェーズ 4: フォルダ作成

1. `~/work/{project-name}/` と `.claude/` を作成
2. `.claude/plan.md` を書き込む
3. `CLAUDE.md` を書き込む
4. 次のステップをユーザーに伝える

## 原則

- 質問は段階的に。前の回答を踏まえて次の質問を絞る
- `CLAUDE.md` は30秒で全体像を掴めるレベルに。詳細は `.claude/plan.md` へ
- 計画はプロジェクト内 `{project}/.claude/plan.md` に置く（グローバル plans フォルダではない）
